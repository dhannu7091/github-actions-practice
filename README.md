# GitHub Actions Practice

This repository is used to practice and experiment with **GitHub Actions workflows** and understand how CI/CD pipelines work in a real development environment.

The goal is to learn how automation can be used to build, test, and deploy applications using GitHub Actions.

---

## 🚀 What I'm Practicing

- Creating GitHub Actions workflows
- Understanding CI/CD pipelines
- Working with YAML workflow files
- Running jobs on GitHub-hosted runners
- Automating build, test, and deployment tasks
- Triggering workflows on push and manual events

---

## 📂 Repository Structure

.github/
└── workflows/
├── hello.yml
├── cicd.yml
└── portfolio-deploy.yml

Each workflow demonstrates different concepts of GitHub Actions.

---

## ⚙️ Workflows in This Repository

### 1️⃣ Hello Workflow
A simple workflow that runs on push and prints messages using GitHub Actions.

Purpose:
- Understand basic workflow structure
- Learn how jobs and steps work

---

### 2️⃣ CI/CD Workflow
A multi-stage pipeline that simulates a CI/CD process.

Stages included:
- Code step
- Build step
- Test step
- Deployment step (environment based)

Purpose:
- Understand job dependencies
- Practice pipeline structure used in DevOps

---

### 3️⃣ Portfolio Deployment Workflow
This workflow deploys a static website using **GitHub Pages**.

Purpose:
- Learn automated deployment
- Practice GitHub Pages deployment using workflows
